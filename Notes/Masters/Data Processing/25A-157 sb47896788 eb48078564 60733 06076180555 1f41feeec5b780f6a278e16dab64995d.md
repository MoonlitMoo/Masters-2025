# Pipeline w/ Yvette Script

Having trouble during the final calibration step. 

- Yvette email
    
    Finally made some time to look at this.  It's an interesting one - it seems the issue is that the phase is actually jumping around a lot more than usual, must have been bad weather or something.  So to calibrate the calibrator, we definitely want to keep solint='int'.  And a phase gaincal with solint='int' works well for all the good spectral windows (<22), however not well for the bad spws.  My feeling is the pipeline probably decided to use a longer solution interval because too many of the solutions were thrown away in the bad spws, but then because of the unusually large phase changes that meant the calibrator wasn't calibrating well.  So, a strategy that I think works:
    
    1. Phase-only gaincal, with solint='int' and combine='spw', but selecting only spws<22 (including the bad ones messes things up). This gives a single, time-dependent solution per antenna per correlation which resemble the non-combined solutions pretty well. You can apply these to the calibrator to check (using spwmap=[32*[0]] to tell it to apply the single solution to all spws) and things look good.
    2. There is a small remnant phase/amp gradient across the band after doing this, which looks like it's time-dependent as well. This would be gotten rid of by doing a spw-dependent gaincal or a bandpass calibration, but in either case the bad spws mess things up. I wanted to just fit a straight line across the gradients across the whole band, which I thought you could do with the BPOLY option in the bandpass task but it turns out that doesn't let you fit across the whole band. So, I wrote some hacky code to do the fitting and put fitted solutions into a bandpass table. It's uploaded to the shared folder and is called fit_bpsols.py (needs to run within casa). You have to have created a bandpass table using all spws and one solution per scan first; it takes the table, grabs the data and fits to it, and puts it into a new table.
    
    At the moment the fitted solutions are being influenced by the bad points, but the advantage of doing it this way is that you could be quite aggressive with the flagging beforehand, as it wouldn't matter anymore if you lost any spws.  Then to apply to the science field, I would probably just use these two tables rather than creating a new, longer-average phase table.
    

## Hack

- Run the final cal + applycal as normal
- Using finalcalibrator (make sure no flagging), apply phase cal + yvette bpsol
- Apply resulting tables to field 1.
- Note, I forgot that there was only 1 secondary here, could have been easier to yoink commands from fluxscale etc.

## Full casa_pipescript

```python
# This CASA pipescript is meant for use with CASA 6.6.1 and pipeline 2024.1.0.8
context = h_init()
context.set_state('ProjectSummary', 'observatory', 'Karl G. Jansky Very Large Array')
context.set_state('ProjectSummary', 'telescope', 'EVLA')
try:
    hifv_importdata(vis=['25A-157.sb47896788.eb48078564.60733.06076180555.ms'])
    hifv_flagdata(hm_tbuff='1.5int', fracspw=0.01, intents='*POINTING*,*FOCUS*,*ATMOSPHERE*,*SIDEBAND_RATIO*, *UNKNOWN*, *SYSTEM_CONFIGURATION*, *UNSPECIFIED#UNSPECIFIED*')
    hifv_vlasetjy()
    hifv_priorcals(show_tec_maps=False)
    hifv_aoflagger(flag_target='"0137+331=3C48"', aoflagger_file='0137+331=3C48.lua')
    hifv_testBPdcals()
    hifv_semiFinalBPdcals()
    hifv_aoflagger(flag_target='J0424+0204', aoflagger_file='J0424+0204.lua')
    hifv_solint()
    hifv_fluxboot()
    hifv_finalcals()
    # Get a bandpass for later, prior to flagging to not kill scan spws.
    bandpass(vis='finalcalibrators.ms', caltable='testphase_inf_b_allspw.b',
        field='1', combine='', minsnr=5.0, bandtype='B', parang=True)
    hifv_aoflagger(flag_target='J0424+0204', aoflagger_file='J0424+0204-corrected.lua', use_corrected=True)
    # Now fix the secondary calibrator
    # Step 1, phase solution for the good spw.
    gaincal(vis='finalcalibrators.ms',
        caltable='final_field1_gain_good_spw_avged/',
        field='1', spw='0~21', solint='int', combine='spw', preavg=-1.0,
        refant='ea15,ea20,ea16,ea12,ea27,ea26,ea25,ea02,ea17,ea19,ea08,ea07,ea01,ea06,ea14,ea09,ea03,ea22,ea28,ea04,ea18,ea11,ea13,ea23,ea24',
        refantmode='flex', minblperant=4, minsnr=3.0, solnorm=False, gaintype='G',
        calmode='p', parang=True)
    # Step 2, fit the gradient solutions
    execfile('fit_bpsols.py')
    # Step 3 Apply calibration to secondary and actual
    applycal( vis='25A-157.sb47896788.eb48078564.60733.06076180555.ms', field='1,2',
             gaintable=['25A-157.sb47896788.eb48078564.60733.06076180555.ms.hifv_priorcals.s4_2.gc.tbl',
                        '25A-157.sb47896788.eb48078564.60733.06076180555.ms.hifv_priorcals.s4_3.opac.tbl',
                        '25A-157.sb47896788.eb48078564.60733.06076180555.ms.hifv_priorcals.s4_4.rq.tbl',
                        '25A-157.sb47896788.eb48078564.60733.06076180555.ms.hifv_priorcals.s4_6.ants.tbl',
                        '25A-157.sb47896788.eb48078564.60733.06076180555.ms.hifv_finalcals.s11_2.finaldelay.tbl',
                        '25A-157.sb47896788.eb48078564.60733.06076180555.ms.hifv_finalcals.s11_4.finalBPcal.tbl',
                        '25A-157.sb47896788.eb48078564.60733.06076180555.ms.hifv_finalcals.s11_5.averagephasegain.tbl',
                        'final_field1_gain_good_spw_avged',
                        'bptable_fitted.b'],
             gainfield=['', '', '', '', '', '', '', '', ''],
             interp=['', '', '', '', '', 'linear,linearflag', '', '', ''],
             spwmap=[[], [], [], [], [], [], [], 32*[0], []],
             calwt=[False, False, False, False, False, False, False, False, False],
             parang=True, applymode='calflagstrict', flagbackup=True)
    # Back to regular pipeline
    hifv_aoflagger(flag_target='A478', aoflagger_file='A478.lua', use_corrected=True)
    hifv_statwt()
    hifv_plotsummary()
    hifv_exportdata()
finally:
    h_save()
```
