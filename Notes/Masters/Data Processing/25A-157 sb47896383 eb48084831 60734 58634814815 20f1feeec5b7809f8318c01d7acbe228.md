# Pipeline part 1

```python
# This CASA pipescript is meant for use with CASA 6.6.1 and pipeline 2024.1.0.8
context = h_init()
context.set_state('ProjectSummary', 'observatory', 'Karl G. Jansky Very Large Array')
context.set_state('ProjectSummary', 'telescope', 'EVLA')
try:
    hifv_importdata(vis=['25A-157.sb47896383.eb48084831.60734.58634814815.ms'])
    hifv_flagdata(hm_tbuff='1.5int', fracspw=0.01, intents='*POINTING*,*FOCUS*,*ATMOSPHERE*,*SIDEBAND_RATIO*, *UNKNOWN*, *SYSTEM_CONFIGURATION*, *UNSPECIFIED#UNSPECIFIED*')
    hifv_vlasetjy()
    hifv_priorcals(show_tec_maps=False)
    hifv_aoflagger(flag_target='0', aoflagger_file='1331+305=3C286.lua')
    hifv_testBPdcals()
    hifv_semiFinalBPdcals()
    hifv_aoflagger(flag_target='1', aoflagger_file='J1616+0459.lua')
    hifv_aoflagger(flag_target='3', aoflagger_file='J1719+1745.lua')
    hifv_solint()
    hifv_fluxboot()
finally:
    h_save()
```

## Playing with J1616

There was a problem with ~6-7 spw being killed off in J1616+0459 during `fluxboot`. This is due to the normalised gains being really extreme in the bad channels due to RFI. This means when the clipping occurs, it just deletes them. The following is some test code I used to tune the aoflagger to minimise flagging while also maximising remaining spw.

Getting the normalised values

```bash
gaincal(vis='calibrators.ms', caltable='test_fluxphaseshortgaincal.g',
	field='1', spw='', solint='int', combine='scan', refant='ea15,ea20,ea16,ea26,ea25,ea02,ea27,ea17,ea08,ea19,ea07,ea01,ea06,ea14,ea09,ea03,ea22,ea28,ea12,ea04,ea18,ea11,ea13,ea23,ea24,ea10',
	minsnr=3.0, solnorm=False, gaintype='G', calmode='p', parang=True)
gaincal(vis='calibrators.ms', caltable='test_fluxflag.g', field='1', spw='', 
	selectdata=False, solint='222.0s', combine='scan', refant='ea15,ea20,ea16,ea26,ea25,ea02,ea27,ea17,ea08,ea19,ea07,ea01,ea06,ea14,ea09,ea03,ea22,ea28,ea12,ea04,ea18,ea11,ea13,ea23,ea24,ea10', 
	minsnr=5.0, solnorm=True, gaintype='G', calmode='ap', 
	gaintable=['test_fluxphaseshortgaincal.g'], parang=True)
```

clipping

```bash
flagdata(vis='test_fluxflag.g', mode='clip', correlation='ABS_ALL', 
	clipminmax=[0.9, 1.1], datacolumn='CPARAM', clipoutside=True, 
	action='apply', flagbackup=False)
```

# Pipeline part 2

```bash
# This CASA pipescript is meant for use with CASA 6.6.1 and pipeline 2024.1.0.8
context = h_init()
context.set_state('ProjectSummary', 'observatory', 'Karl G. Jansky Very Large Array')
context.set_state('ProjectSummary', 'telescope', 'EVLA')
try:
    hifv_importdata(vis=['25A-157.sb47896383.eb48084831.60734.58634814815.ms'])
    hifv_flagdata(hm_tbuff='1.5int', fracspw=0.01, intents='*POINTING*,*FOCUS*,*ATMOSPHERE*,*SIDEBAND_RATIO*, *UNKNOWN*, *SYSTEM_CONFIGURATION*, *UNSPECIFIED#UNSPECIFIED*')
    hifv_vlasetjy()
    hifv_priorcals(show_tec_maps=False)
    hifv_aoflagger(flag_target='0', aoflagger_file='1331+305=3C286.lua')
    hifv_testBPdcals()
    hifv_semiFinalBPdcals()
    hifv_aoflagger(flag_target='1', aoflagger_file='J1616+0459.lua')
    hifv_aoflagger(flag_target='3', aoflagger_file='J1719+1745.lua')
    hifv_solint()
    hifv_fluxboot()
    hifv_finalcals()
    hifv_applycals()
    hifv_aoflagger(flag_target='2', aoflagger_file='A2204.lua', use_corrected=True)
    hifv_aoflagger(flag_target='4', aoflagger_file='RXJ1720.1+2638.lua', use_corrected=True)
    hifv_statwt()
    hifv_plotsummary()
    hifv_exportdata()
finally:
    h_save()
```

## Checks

- Bandpass amp: Looks good.
- Short phase cal: Definitely a subset of bad solutions, but mostly localised and few comparatively.
- Final amp cal: There’s a few points here and there that are bad. Vast majority are good.
- Final phase cal: Handful of outliers, but looks good overall.
